# ai-keiei-sanbo プロジェクト

AI経営参謀プロジェクト。

## Git運用ルール

### 基本方針
- コードを変更するたびに、必ずGitHubにプッシュすること
- コミットは変更の意図が伝わるメッセージで行うこと
- `main` ブランチへの直接プッシュを基本とする

### リモートリポジトリ
- URL: https://github.com/test-cloud-2026/ai-keiei-sanbo.git

### コミット・プッシュの手順
1. ファイルを変更・追加したら即座にステージング
2. 変更内容を簡潔に表すコミットメッセージを付けてコミット
3. 毎回 `git push origin main` でGitHubにプッシュ

```bash
git add <変更ファイル>
git commit -m "変更内容の説明"
git push origin main
```

### コミットメッセージの書き方
- 新機能追加: `add: 機能名`
- 既存機能の更新: `update: 変更内容`
- バグ修正: `fix: 修正内容`
- ドキュメント更新: `docs: 変更内容`

## 開発ガイドライン

- 不要なコメントは書かない（コードが自明な場合）
- セキュリティに配慮したコードを書く（SQLインジェクション、XSS等を避ける）
- シンプルで読みやすいコードを優先する
