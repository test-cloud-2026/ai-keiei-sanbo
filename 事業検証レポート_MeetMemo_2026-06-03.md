# 事業検証レポート：MeetMemo
**作成日：2026-06-03**

> 会議録音をAIが自動で議事録化し、タスクを担当者に通知するSaaS「MeetMemo」の事業仮説を検証する。

---

## 1. 課題仮説の検証

### この課題は実際に多くの人が抱えているか

**結論：課題は明確に実在し、かつ解決されていない。**

| 調査データ | 数値 |
|---|---|
| ビジネスパーソンが議事録作成に費やす時間 | **平均 320時間／年**（ASCII調査） |
| 議事録作成に「負担を感じる」と回答した割合 | **67%** |
| AIサポートツールの使用を「希望する」割合 | **70%** |
| 実際に議事録作成DXが進んでいる現場の割合 | **わずか 1.4%** |
| 会議への参加必要性に疑問を感じたことがある割合 | **88.9%** |
| 週に数回「会議で他業務が圧迫される」と感じる割合 | **42.8%** |

**課題を裏付ける具体的データ・事例**

- **工数の実態**：1時間の会議に対し、会議後の議事録作成に **2〜3.5時間** かかるのが従来の標準（Ragate株式会社、情報システム部門・DX推進室ビジネスパーソン506名調査、2025年12月）。
- **AI導入後の改善効果**：AI議事録ツール導入後は確認・修正時間が **15〜30分** に短縮。1会議あたり約2時間の工数削減が試算される。
- **コスト試算例**：2時間削減 × 5会議/週 × 50週 × 人件費5,000円/h ＝ **年間250万円／人** の削減効果。
- **活用されない議事録問題**：調査では「議事録を作成しても活用されない」と感じるビジネスパーソンが **9割弱** に上り（tldx Solutions調べ, 2025年12月）、単なる文字起こしを超えた「タスクへの接続」に対するニーズが存在する。

**課題仮説の評価**：需要は高く、ギャップ（希望70% vs 導入1.4%）が大きい。"録音→議事録"だけでなく"タスク通知まで一気通貫"という付加価値軸で勝負できる余地が十分ある。

## 2. 市場規模の推定

### 国内・海外の市場規模（TAM・SAM・SOMの概算）

```
TAM（到達可能な全体市場）
  グローバルAI会議アシスタント市場
  2025年：約 3,500億円（$3.5B × 150円換算）
  2034年予測：約 4.4兆円（CAGR 25.62%）
  ↓
SAM（獲得可能な市場）
  国内AI議事録・議事録SaaS市場
  2026年：約 380億円（前年比+42%）
  ← 対象：国内企業のうち会議録音・議事録を業務利用する中小〜大企業
  ↓
SOM（現実的な初期獲得市場）
  ローンチ後3年で国内市場の1〜3%を目標
  = 約 4〜11億円 ARR
  ← SMB（従業員10〜300名）5,000社 × 月額2万円/社 = 12億円/年 が現実的上限
```

### 市場の成長トレンド

| 指標 | 数値 |
|---|---|
| グローバルAI Meeting Assistant市場 CAGR（2025〜2034） | **25.62%** |
| 国内AI議事録ツール市場 前年比成長率（2026年） | **+42%** |
| 国内AIシステム市場 CAGR（2024〜2029） | **25.6%** |
| 国内100名以上企業でのAI議事録ツール導入率（2026年） | **約35%** |

**トレンドの質的変化**：市場の重心は「録音して議事録を作る」ツールから「対話データを起点に業務を自動化する」AIエージェントプラットフォームへ移行中。タスク自動抽出・担当者通知機能はこのトレンドの中心に位置する。

## 4. 差別化ポイントの整理

### 競合との比較で自社（MeetMemo）が勝てる軸

---

#### 差別化軸① ：「議事録で終わらせない」タスク通知一気通貫

**競合の状況**：Nottaは文字起こし・要約が中心でタスク通知は別途手動作業が必要。Firefliesはタスク抽出機能を持つが英語中心で日本語環境への最適化が不十分。tl;dvはタスク通知統合が弱い。

**MeetMemoの優位性**：
- 会議録音 → AIによる議事録生成 → **「誰が・何を・いつまでに」のタスク自動抽出** → Slack/メールで担当者へ直接通知、という一気通貫フローを日本語で完結させる。
- 競合ツールでは「議事録作成後のタスク共有」に依然として人手が介在しており、ここが最大の未解決ペイン。

---

#### 差別化軸② ：日本語・日本のビジネス環境への深い最適化

**競合の状況**：Fireflies・tl;dvは英語圏中心で日本語認識精度・UIが不十分。Nottaは日本語精度は高いが中国企業であり、データ保存先・セキュリティへの懸念を持つ日本企業が存在する。

**MeetMemoの優位性**：
- 日本語特化モデルによる高精度認識（敬語・業界専門用語・略語に対応）。
- データを**国内サーバー（日本リージョン）に保管**し、ISMSや個人情報保護法への準拠を訴求。
- 日本語UIと日本語サポート体制で中小企業のIT担当者が導入しやすい設計。

---

#### 差別化軸③ ：中小企業（SMB）向けのシンプル・低価格設計

**競合の状況**：Nottaのビジネスプランは月2,500円〜/ユーザーで機能が豊富だが複雑。Firefliesは英語前提でSMBには導入障壁が高い。エンタープライズ向けCopilot等は高額でSMBに不向き。

**MeetMemoの優位性**：
- 設定ゼロで使える「会議参加→録音→通知」のシンプルな3ステップUX。
- **チームプラン：月額1,980円/ユーザー**（年払い時）で小規模チームから導入可能。
- タスク管理ツール（Backlog・Asana等）や社内チャット（Chatwork・LINE WORKS）等、日本のSMBが使うツールへの連携を優先実装。
